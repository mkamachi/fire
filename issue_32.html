Issue #32: FIRE 2025/04/19

ここ2ヶ月のLinux関連の重要ニュースまとめ（2025年2月～4月中旬） 1. セキュリティ関連の主なアップデート 主要ブラウザのLinux版でセキュリティ修正が相次ぐ Google Chrome 135.0.7049.95（Linux版含む）がリリースされ、2件のセキュリティ修正が行われた。 Mozilla Firefox 137.0および137.0.2（ESR版含む）、Thunderbird 137.0.2も公開され、複数の脆弱性が修正された。 Erlang/OTPのSSHサーバーに重大な脆弱性 Erlang/OTPのSSHサーバーに認証不要のリモートコード実行（RCE）脆弱性（CVE-2025-32433）が発見され、全バージョンが影響を受ける。 OpenSSL 3.5が正式リリース 耐量子暗号対応やサーバー側QUIC対応など、セキュリティ強化を含むOpenSSL 3.5がリリースされた。 2. オープンソース・関連ソフトウェアの動向 SoftEther VPNが約2年ぶりにアップデート オープンソースVPN「SoftEther VPN」製品版が久々にアップデートされ、「TunnelCrack」対策や機能強化、不具合修正が行われた。 Oracle VM VirtualBox v7.0系がサポート終了 仮想化ソフトウェア「Oracle VM VirtualBox」v7.0系のサポートが終了し、今後は新バージョンへの移行が推奨されている。 3. 脆弱性管理体制の国際的な変化 CVEプログラムの運営体制が財団化 世界中の脆弱性情報を管理するCVE（Common Vulnerabilities and Exposures）プログラムが「CVE Foundation」に移管され、米国一国依存から脱却する動きが進んでいる。 4. その他注目トピック Apple、TLSサーバー証明書の有効期間を大幅短縮へ AppleがTLSサーバー証明書の最大有効期間を47日間に短縮する提案が承認され、202[6](https://www.st.ryukoku.ac.jp/~kjm/security/memo/)年3月から段階的に移行予定。Linuxサーバー運用者にも影響が及ぶ可能性がある。 まとめ この2ヶ月間、Linux関連ではセキュリティアップデートや重要な脆弱性の発見、主要オープンソースソフトウェアのアップデートが相次ぎました。特にErlang/OTPのSSHサーバー脆弱性や、OpenSSL 3.5のリリース、CVE運営体制の変化は、Linuxユーザーや管理者にとって注目すべき動きです。加えて、証明書運用の自動化やサイクル短縮も今後の課題となるでしょう。
